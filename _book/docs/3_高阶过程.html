高阶过程

Tips: 所有的图示的计算过程不是非常严谨

先看一个求和一组整数的求和公式： ∑i=abi \sum_{i=a}^b i ​i=a​∑​b​​i 上述公式抽象成 代码①：

(define (sum-sq a b)
    (if (> a b)
        0
        (+ (square a) (sum-sq (1+ a) b))))

简单描述下 (sum-int 1 4) 的计算过程（过程未写完整）：

graph TD A["(sum-int 1 4)"] --> B["(+ 1 (sum-int 2 4)"] B --> C["(+1 (+ 2 (sum-int 3 4)))"] C --> D["(+1 (+ 2 (+ 3 (sum-int 4 4)))"] D --> E["(+1 (+ 2 (+ 3 (+ 4 (sum-int 5 4))))"] E --> F["(+1 (+ 2 (+ 3 (+ 4 0)))"]

再看一个数学公式： ∑i=abi2 \sum_{i=a}^b i^{2} ​i=a​∑​b​​i​2​​ 上述公式抽象成 代码②:

(define (sum-sq a b)
    (if (> a b)
        0
        (+ （square a) (sum-sq (1+ a) b))))

再看一个莱布尼茨公式（省略数学公式书写），用来求 π / 8 的值。 代码③:

(define (pi-sum a b)
    (if (> a b)
        0
        (+ (/ 1 (* a (+ a 2))) 
            (sum-sq (+ a 4) b))))

我们看到 代码①、代码②、代码③ 的大部分过程都是相同的，只有一些细微的差别，我们来对此进行抽象 (伪代码)。

(define (<name> a b)
    (if (> a b)
        0
        (+ (<term> a)
            (<name> (<next> a) b))))

我们将伪代码变成真实的代码：

(define (sum term a next b)
    (if (> a b)
        0
        (+  (term a) 
            (sum term (next a) next b)
        )
    )
)

我们有四个参数

  • term ：过程
  • a：下界的下标
  • next：获得下一个下标的方法
  • b：上界

这是我们将 代码① 重写：

(define (sum-int a b)
    (define (indent x) x)
    (define (next y) (+ y 1))
    (sum indent a next b)
)

可以看到，函数内部定义了两个计算过程，这两个过程可以进行替换。

我们再将 代码② 重写：

(define (sum-square a b)
    (define (square x) (* x x))
    (sum square a 1+ b)
)
